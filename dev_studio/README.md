# 💻 SaleH Dev Studio — بيئة التطوير المتكاملة

> **VS Code في المتصفح + ذكاء اصطناعي محلي = بيئة تطوير احترافية بالكامل**
>
> **تنبيه تشغيلي:** هذا المسار اختياري للتطوير وليس جزءاً من `docker-compose.yml` التشغيلي الأساسي الحالي. لا يتم تشغيله افتراضياً إلا عبر ملفه المنفصل `dev_studio/docker-compose.dev-studio.yml`.
>
> **قاعدة قرار:** إذا لم يتم تشغيل `dev_studio/docker-compose.dev-studio.yml` صراحة، فافترض أن `Code Server` غير متاح ولا تعتمد عليه أثناء اتخاذ القرار التشغيلي.

---

## نظرة عامة

هذا المجلد يحتوي على كل ما يلزم لتشغيل **SaleH Dev Studio**، وهي بيئة تطوير متكاملة (IDE) تعمل بالكامل من خلال المتصفح، ومصممة خصيصاً لمشروع SaleHSaaS. تدمج هذه البيئة بين قوة VS Code، والذكاء الاصطناعي المحلي عبر Ollama، وأدوات التطوير الحديثة لتقديم تجربة تطوير سلسة وفعالة.

## المكونات الرئيسية

| المكوّن | الوصف |
| :--- | :--- |
| **`docker-compose.dev-studio.yml`** | ملف الإعداد لتشغيل مسار Dev Studio الاختياري المبني على Code Server بشكل منفصل عن runtime الأساسي. |
| **`config/`** | يحتوي على ملفات الإعدادات المخصصة لـ VS Code (`vscode-settings.json`) وامتداد Continue.dev (`continue-config.json`). |
| **`scripts/`** | يحتوي على سكريبتات الأتمتة لتسهيل المهام المتكررة. |
| **`workspace/`** | مجلد العمل الافتراضي داخل Dev Studio / Code Server عند تشغيله صراحة. |

## كيفية التشغيل

1.  **تأكد من وجود ملف `.env` في المجلد الرئيسي للمشروع** وتعيين المتغيرات التالية:

    ```env
    CODE_SERVER_PASSWORD=كلمة_مرور_قوية
    DEV_STUDIO_PORT=8080
    ```

2.  **شغّل Dev Studio باستخدام Docker Compose:**

    ```bash
    # من المجلد الرئيسي للمشروع
    docker compose -f dev_studio/docker-compose.dev-studio.yml up -d
    ```

3.  **افتح المتصفح** وانتقل إلى `http://localhost:8080` (أو المنفذ الذي حددته).

4.  **أدخل كلمة المرور** التي قمت بتعيينها في ملف `.env`.

> ملاحظة: تشغيل هذا المسار لا يجعله جزءًا من التشغيل الحي الأساسي للمشروع؛ يبقى مسار تطوير اختياري مستقل.

## الميزات

- **بيئة موحدة:** نفس بيئة التطوير لجميع أفراد الفريق.
- **تكامل مع Docker:** إدارة الحاويات والخدمات مباشرة من داخل VS Code.
- **مدعوم بالذكاء الاصطناعي:**
    - **إكمال تلقائي ذكي** للكود.
    - **محادثة مع نماذج AI** لشرح، مراجعة، وتحسين الكود.
    - **أوامر مخصصة** لتوليد الاختبارات، ملفات Docker، واجهات API، والمزيد.
- **وصول كامل للمشروع:** تعديل أي ملف في المشروع مباشرة من المتصفح.
- **طرفية مدمجة:** تنفيذ الأوامر، تشغيل السكريبتات، ومراقبة السجلات.
