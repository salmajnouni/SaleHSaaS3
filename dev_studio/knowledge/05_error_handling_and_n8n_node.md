# معالجة الأخطاء وعقدة n8n الداخلية

## معالجة الأخطاء (Error Handling)

n8n يتيح إنشاء سير عمل مخصص للتعامل مع الأخطاء. عند فشل أي تنفيذ، يُشغَّل سير العمل المخصص للأخطاء تلقائياً.

### خطوات الإعداد

يجب إنشاء سير عمل جديد يبدأ بعقدة **Error Trigger**، ثم ربطه بسير العمل الرئيسي من **Workflow Settings ← Error Workflow**.

### بيانات الخطأ المتاحة

عند تشغيل سير عمل الأخطاء، تصل إليه البيانات التالية:

```json
[
  {
    "execution": {
      "id": "231",
      "url": "https://n8n.example.com/execution/231",
      "error": {
        "message": "رسالة الخطأ",
        "stack": "تفاصيل الخطأ"
      },
      "lastNodeExecuted": "اسم العقدة التي فشلت",
      "mode": "manual"
    },
    "workflow": {
      "id": "1",
      "name": "اسم سير العمل"
    }
  }
]
```

### عقدة Stop And Error

تُستخدم لإيقاف سير العمل وإطلاق خطأ مخصص في أي مرحلة، مما يُشغّل سير عمل الأخطاء.

## عقدة n8n الداخلية (n8n Node)

هذه العقدة تتيح التحكم الكامل في n8n من داخل سير العمل نفسه، عبر استدعاء n8n API داخلياً.

### العمليات المتاحة

**على سير العمل (Workflow):**

| العملية | الوصف |
|---------|-------|
| `Create` | إنشاء سير عمل جديد من كائن JSON |
| `Get` | جلب بيانات سير عمل بمعرّفه |
| `Get Many` | جلب قائمة بكل سير العمل |
| `Update` | تحديث سير عمل موجود |
| `Delete` | حذف سير عمل |
| `Publish (Activate)` | تفعيل سير عمل |
| `Deactivate` | إلغاء تفعيل سير عمل |

**على التنفيذات (Execution):**

| العملية | الوصف |
|---------|-------|
| `Get` | جلب تفاصيل تنفيذ بمعرّفه |
| `Get Many` | جلب قائمة التنفيذات مع فلترة بالحالة (Error/Success/Waiting) |
| `Delete` | حذف تنفيذ |

### مثال: إنشاء سير عمل جديد من داخل n8n

```json
{
  "operation": "create",
  "workflowObject": {
    "name": "سير عمل جديد",
    "nodes": [],
    "connections": {},
    "settings": {"executionOrder": "v1"}
  }
}
```
