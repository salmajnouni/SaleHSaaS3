# بنية JSON في n8n

كل سير عمل في n8n هو ملف JSON. هذا الملف يصف كل شيء: العقد، إعداداتها، والوصلات بينها. فهم هذه البنية هو مفتاح توليد سير عمل صحيح.

## الهيكل العام

```json
{
  "name": "اسم سير العمل",
  "nodes": [],
  "connections": {},
  "settings": {"executionOrder": "v1"}
}
```

- **`name`**: اسم سير العمل الذي يظهر في واجهة n8n.
- **`nodes`**: مصفوفة تحتوي على كل العقد في سير العمل.
- **`connections`**: كائن يصف الوصلات بين العقد.
- **`settings`**: إعدادات عامة لسير العمل.

## بنية العقدة (Node)

كل عقدة في مصفوفة `nodes` هي كائن له هذه الحقول:

```json
{
  "id": "uuid-فريد-للعقدة",
  "name": "اسم العقدة",
  "type": "نوع العقدة",
  "typeVersion": 1.1,
  "position": [250, 300],
  "parameters": {}
}
```

- **`id`**: معرّف فريد لكل عقدة.
- **`name`**: الاسم الذي يظهر على العقدة في الواجهة.
- **`type`**: نوع العقدة، مثل `n8n-nodes-base.scheduleTrigger`.
- **`typeVersion`**: إصدار نوع العقدة.
- **`position`**: مصفوفة تحتوي على إحداثيات `[x, y]` لموقع العقدة على اللوحة.
- **`parameters`**: كائن يحتوي على كل إعدادات العقدة.

## بنية الوصلات (Connections)

كائن `connections` يصف كيف تتدفق البيانات. المفتاح هو **اسم العقدة المصدر**، والقيمة هي كائن يصف الوصلات الخارجة منها.

```json
"connections": {
  "اسم العقدة المصدر": {
    "main": [[{"node": "اسم العقدة الهدف", "type": "main", "index": 0}]]
  }
}
```

- **`main`**: نوع المخرج من العقدة المصدر (عادة `main`).
- **`node`**: اسم العقدة الهدف التي تستقبل البيانات.

## بنية البيانات (Data Structure)

البيانات التي تتدفق بين العقد هي دائماً **مصفوفة من الكائنات**. كل كائن في المصفوفة يمثل عنصراً واحداً.

```json
[
  {
    "json": {
      "field1": "value1",
      "field2": 123
    }
  },
  {
    "json": {
      "field1": "value2",
      "field2": 456
    }
  }
]
```

- كل عنصر يجب أن يكون له مفتاح `json` يحتوي على البيانات الفعلية.
- للبيانات الثنائية (ملفات)، استخدم مفتاح `binary` بدلاً من `json`.
