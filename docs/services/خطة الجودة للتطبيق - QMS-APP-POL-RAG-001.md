## 6) سياسات RAG الأساسية
### 6.1 سياسة مصادر المعرفة
- لا يتم إدخال أي مصدر غير معتمد.
- كل مصدر يجب أن يحمل Metadata: المصدر، التاريخ، المالك، الغرض.
- أي مصدر غير موثق يسجل Policy Violation.

### 6.2 سياسة تقليل البيانات (PDPL)
- جمع الحد الأدنى فقط من البيانات.
- تمييز البيانات الحساسة بعلامة واضحة.
- منع استخدام البيانات الحساسة خارج الغرض المعتمد.

### 6.3 سياسة جودة الإدخال
- أي ملف يمر بوابة جودة قبل الفهرسة.
- الفشل يذهب إلى knowledge_failed مع سبب الخطأ.
- ممنوع الفشل الصامت.

### 6.4 سياسة نزاهة الاسترجاع
- الإجابة يجب أن تكون مبنية على دليل مسترجع.
- عند ضعف الدليل، يتم إرجاع تنبيه بعدم كفاية الأدلة.
- اختيار collection أو API path أو أي مسار تشغيلي يعتمد على المرجع الحاكم الحالي، وليس على وصف تاريخي داخل وثيقة أقدم.
- إعدادات retrieval (top-k/threshold/collection) تُحفظ في سجل تغيير.

### 6.5 سياسة الإشراف البشري
- أي مخرج عالي الحساسية يحتاج اعتماد مالك القرار قبل الإرسال الخارجي.

### 6.6 سياسة التتبع والسجلات
- كل عملية Query/Response يجب أن تتضمن: request id، timestamp، المصادر المرجعية، حالة القرار.
- السجلات تعتبر أدلة جودة وتخضع لضبط 7.5.

### 6.7 سياسة الاحتفاظ والحذف
- تعريف دورة احتفاظ لكل نوع سجل.
- حذف السجلات المنتهية يتم عبر إجراء موثق مع سجل حذف.

### 6.8 سياسة أمن الوصول
- صلاحيات مبنية على الدور.
- أي خرق أو تسريب يرفع مباشرة بعلامة GRC-CRITICAL.
